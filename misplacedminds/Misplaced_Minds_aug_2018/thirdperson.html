10 Easy Steps To Create a Third Person Camera In UDK:




Creating a third person camera in UDK may seem to be a very tough task for an absolute beginner. In fact, it isn't a very difficult task. If you understand what is going on, it is fun changing the camera views and settings. In this tutorial, you'll not only learn how to set up a third person camera but also adjust the co-ordinates (camera distance from player).

What we are going to do:


1. Modify the UC scripts and.ini files to change the camera from default first person to third person.
2. This method is applicable for setting up any camera view (including All in One camera)

Important Note-1:

The purpose of this tutorial is to help you set up the camera view into third person. To make it as simple as possible, very less explanation of the code is given.

Important Note-2:


1. To be able to view the changes, the levels should be saved with the prefix "UDN" (UDN-Demo.udk)
2. Go to View Menu ->World Properties -> Game Type and set "Game Type for PIE" to UDNGame. (This cannot be edited now. Remember this point till the end of this tutorial.After completing the steps given below successfully, you should use this.)


Getting Started:

1. Create a folder named "magneticthumbs" (You can use whatever name you want).
2. Given are the codes for three files UDNPawn.uc, UDNPlayerController.uc and UDNGame.uc
3. Create another folder called "Classes" inside "magneticthumbs" .
4. Open Notepad and paste these codes and save them with respective titles.

Adjusting the Camera Distance:


1. Open the UDNPawn.uc and go to the last line of the code.
2. You can see:

defaultproperties
{
}

3. Add this code between the braces

CamOffset=(X=15.0,Y=40.0,Z=-20.0)

4. Adjust the co-ordinates for changes.



Notes:
1. To view the third person camera view, change the game type in the world properties (Refer to the Important Note -2 at the beginning of this           tutorial.)
2. The above code can also be obtained from "The Camera Technical Guide-UDK.com".



3. While navigating through UDK folders, UDK-xxxx-xx refers to the version of UDK
4. The code given above should be properly copied and saved.You may have to change the file type to save to 'all files' from 'text files'



5. If you change the class name from magneticthumbs to any custom name, don't forget to change it in all the required files.
6. If you come across any errors or warnings in the compilation, try again.
7. Don't forget to backup the files before replacing. We are not responsible for any damage or information loss caused.
